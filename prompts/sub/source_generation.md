# Sub-Agent Prompt: Source List Generation

## Task

Generate `cmake/sources.cmake` for V8 version `{V8_VERSION}` by parsing the V8 BUILD.gn files.

## Input

- `V8_SOURCE_DIR`: Path to the V8 source tree (e.g., `E:/work/v8/v8-src`)
- `V8_VERSION`: Version string (e.g., `13.6.233.17`)
- `FEATURES_JSON`: Path to the version features manifest
- `REFERENCE_SOURCES`: Path to an existing sources.cmake from a nearby version (for structure reference)

## Approach

### Step 1: Parse BUILD.gn

Read `{V8_SOURCE_DIR}/BUILD.gn` (the main file, ~5000-7000 lines). Extract source file lists for these GN targets:

| GN Target | CMake Target |
|-----------|-------------|
| `v8_libbase` | `v8_libbase` |
| `v8_libplatform` | `v8_libplatform` |
| `v8_libsampler` | `v8_libsampler` |
| `v8_compiler` (or `v8_compiler_sources`) | `v8_compiler` |
| `v8_base_without_compiler` | `v8_base_without_compiler` |
| `v8_initializers` | `v8_initializers` |
| `v8_init` | `v8_init` |
| `v8_bigint` | `v8_bigint` |
| `v8_heap_base` (or `cppgc_base`) | `v8_heap_base` |
| `v8_cppgc` (or `cppgc`) | `v8_cppgc` |
| `torque-compiler` or `torque` | `torque` |
| `mksnapshot` | `mksnapshot` |
| `bytecode_builtins_list_generator` | `bytecode_builtins_list_generator` |
| `d8` | `d8` |

### Step 2: Filter for Windows x64

When extracting sources, apply these GN conditionals:
- `is_win = true`
- `target_cpu = "x64"` / `v8_target_cpu = "x64"`
- `is_posix = false`, `is_linux = false`, `is_mac = false`, `is_android = false`
- `v8_enable_webassembly = true` (unless features say otherwise)
- `v8_enable_i18n_support = true` (unless features say otherwise)

Include sources from matching `if` blocks. Exclude sources from non-matching platform blocks.

### Step 3: Verify files exist

For every source file in the generated list, verify it exists at `{V8_SOURCE_DIR}/{path}`. Remove any that don't exist and flag them.

### Step 4: Cross-reference with filesystem

Scan key directories for .cc files that might be missing from BUILD.gn extraction:
- `src/builtins/` — especially platform-specific `*-win64*`, `*-x64*`
- `src/compiler/` and `src/compiler/backend/x64/`
- `src/codegen/x64/`
- `src/execution/`
- `src/wasm/` (if wasm enabled)

### Step 5: Output

Generate `cmake/sources.cmake` using the `v8_src()` helper macro format:

```cmake
# Auto-generated for V8 {V8_VERSION} — do not edit manually
# Generated by scripts/generate_sources.py on {DATE}

macro(v8_src VAR)
  set(_files "")
  foreach(_f ${ARGN})
    list(APPEND _files "${V8_ROOT}/${_f}")
  endforeach()
  set(${VAR} ${_files})
endmacro()

# ---- v8_libbase ----
v8_src(V8_LIBBASE_SOURCES
  src/base/file1.cc
  src/base/file2.cc
  ...
)

# ---- v8_compiler ----
v8_src(V8_COMPILER_SOURCES
  src/compiler/file1.cc
  ...
)
```

## Validation Checklist

Before outputting, verify:
- [ ] Every .cc file in the list exists on disk
- [ ] No test files (test/, testing/) are included in library targets
- [ ] Platform-specific files are only for win/x64
- [ ] The d8 sources include `src/d8/d8.cc`, `src/d8/d8-console.cc`, etc.
- [ ] If Maglev is enabled, `src/maglev/` files are in v8_base_without_compiler
- [ ] If WASM is enabled, `src/wasm/` files are included
- [ ] The total file count is reasonable (compare to reference: 14.3 has ~1100 files total)

## Error Handling

- If BUILD.gn cannot be parsed for a target, fall back to filesystem scanning of the corresponding `src/` directory
- If a target doesn't exist in this version (e.g., no Maglev in 9.4), skip it and note in output
- Report any ambiguities for human review
